Overview
Architecture Summary

The Meridian inventory dashboard is a two-tier web application consisting of a Vue 3 single-page application served by Vite on port 3000 and a FastAPI backend served on port 8001. All persistent state is currently held in memory as JSON data loaded at server startup. The following diagram and reference tables document the as-found architecture inherited from the previous vendor.

Layer Detail
Component Reference

Detailed breakdown of each layer's responsibilities, key files, and notable design decisions observed in the inherited codebase.

Frontend — Views & Router

  • Framework: Vue 3 (Composition API) + Vite
  • Router: Vue Router, 8 named routes
  • Dashboard.vue — KPI summary, top items
  • Inventory.vue — stock levels by warehouse
  • Orders.vue — order list with status filtering
  • Demand.vue — forecast charts
  • Spending.vue — cost analysis, trend charts
  • Reports.vue — quarterly and monthly reports
  • Restocking.vue — purchase order recommendations
  • Backlog.vue — pending work items

Frontend — Shared Components

  • FilterBar.vue — global filter controls (period, location, category, status); writes to useFilters singleton
  • LanguageSwitcher.vue — toggles locale (en / ja / pl); persists to localStorage
  • ProfileMenu.vue — user avatar dropdown, triggers modals
  • ProfileDetailsModal.vue — editable user profile overlay
  • TasksModal.vue — task management overlay backed by /api/tasks

Frontend — Composables (Shared State)

  • useFilters.js — module-level refs: selectedPeriod, selectedLocation, selectedCategory, selectedStatus; shared across all views without Pinia/Vuex
  • useI18n.js — loads locale files (en.js, ja.js, pl.js), exposes t(key) translation helper, persists choice to localStorage
  • useAuth.js — holds session/user state, consumed by ProfileMenu and route guards

API Client — api.js

  • Centralised Axios instance, base URL http://localhost:8001/api
  • All requests serialise active filters via URLSearchParams — no per-view filter logic
  • getCurrentFilters() reads from useFilters singleton at call time
  • Each view calls a dedicated function (e.g., fetchInventory(), fetchOrders())
  • Responses are plain JSON arrays/objects; no client-side caching layer

Backend — FastAPI (main.py)

  • Python 3 / FastAPI / Uvicorn on port 8001
  • CORS middleware enabled for localhost origins
  • All data loaded into module globals at startup via mock_data.py
  • apply_filters() utility filters by warehouse, category, status
  • filter_by_month() utility filters time-series data by month string
  • No authentication middleware on API routes (internal use assumed)
  • No database — restart resets any writes (tasks, purchase orders)

Data Layer — In-Memory JSON

  • Source JSON files read once at server startup
  • inventory_items — warehouse stock records
  • orders — historical and open orders
  • demand_forecasts — SKU-level demand predictions
  • backlog_items — unprocessed work queue
  • spending_summary / monthly_spending / category_spending
  • recent_transactions — ledger-style transaction log
  • purchase_orders — write target for restocking API
  • Note: no persistence — data resets on server restart
Data Flow
Filter Chain — Request Lifecycle

All views share the same filter propagation pattern. A change in FilterBar flows through the singleton composable to every active view simultaneously.

# Step Actor Detail
1 User changes filter FilterBar.vue User selects a warehouse, category, status, or period from the global filter bar
2 Singleton updated useFilters.js Module-level Vue refs (selectedLocation etc.) are updated; reactivity propagates
3 View reacts e.g. Reports.vue Each view watches the relevant filter refs; triggers a fresh API call
4 Request built api.js getCurrentFilters() reads the singleton; parameters appended as URLSearchParams
5 HTTP request Axios → FastAPI GET request to e.g. /api/reports/quarterly?warehouse=SEA&category=Fasteners
6 Server filters main.py apply_filters() and/or filter_by_month() reduce the in-memory dataset
7 Response rendered View component JSON array assigned to reactive data; Vue re-renders the table or chart
